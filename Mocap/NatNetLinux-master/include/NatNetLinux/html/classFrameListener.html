Detailed Description

Thread to listen for MocapFrame data.

Author:
Philip G. Lee

This class listens for MocapFrame data on a given socket. It uses a circular buffer to store the frame data, and provides a thread-safe interface to query for the most recent frames.

std::pair<MocapFrame, struct timespec> FrameListener::pop ( bool *  success = 0) [inline]

Get the latest frame and remove it from the internal buffer. Thread-safe.

This function may block while reading the buffer. success will be false only if there is no more data in the internal buffer.

Parameters:
successinput parameter. If not null, its value is set to true if the return value is valid, and false otherwise.
Returns:
most recent frame/timestamp pair if the internal buffer has data. Otherwise, returns an invalid frame. The timestamp is the result of clock_gettime( CLOCK_REALTIME, ...) when the data is read from the UDP interface.

std::pair<MocapFrame, struct timespec> FrameListener::tryPop ( bool *  success = 0) [inline]

Get the latest frame and remove it from the internal buffer. Thread-safe *non-blocking*.

This function is just like pop(), but never blocks. If you are willing to wait to acquire the lock to read the internal data buffer, then use the blocking version pop(). If you use tryPop(), it will not block, but success will be false if either the buffer is currently being written, *or* if there is no more data.

Parameters:
successinput parameter. If not null, its value is set to true if the return value is valid, and false otherwise.
Returns:
most recent frame/timestamp pair if the internal buffer has data *and* is available for reading immediately. Otherwise, returns an invalid frame. The timestamp is the result of clock_gettime( CLOCK_REALTIME, ...) when the data is read from the UDP interface.
